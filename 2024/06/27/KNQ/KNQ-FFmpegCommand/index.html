关于FFmpegCommand的解释说明

以下是FFmpegCommand类中所有函数的解释，包括每个函数的整体作用以及每个参数的含义。

1. outputStreamFormat(String streamUrl)

整体作用：根据流的URL（streamUrl）返回不同的输出格式参数数组。

参数解释： - streamUrl：流的URL。 - 如果streamUrl以rtsp开头： - "-f", "rtsp"：设置输出格式为RTSP。 - "-pkt_size", "1316"：设置数据包大小为1316字节。 - "-rtsp_transport", "tcp"：指定RTSP传输使用TCP协议。 - 如果streamUrl以rtmp开头： - "-f", "flv"：设置输出格式为FLV。 - "-pkt_size", "1316"：设置数据包大小为1316字节。 - 否则，抛出IllegalArgumentException。

2. convertVideoToStream(String videoPath, String outputStream)

整体作用：将视频文件转换并推流到指定的流媒体服务器。

参数解释： - videoPath：视频文件路径。 - outputStream：输出流的URL。 - XArray.concat方法连接以下数组： - "-hide_banner"：隐藏FFmpeg的启动横幅信息。 - "-re"：以实时模式读取输入流。 - "-i", videoPath：指定输入视频文件。 - "-c:v", "copy"：视频流直接复制，不重新编码。 - "-c:a", "aac"：音频编码格式为AAC。

3. segment(String inputStream, long duration, String workDir)

整体作用：将输入视频流按指定时长切割成多个小段视频文件。

参数解释： - inputStream：输入视频流路径。 - duration：每个视频段的持续时间（秒）。 - workDir：保存分段视频文件的目录。 - "-hide_banner"：隐藏FFmpeg的启动横幅信息。 - "-y"：自动覆盖输出文件。 - "-re"：以实时模式读取输入流。 - "-i", inputStream：指定输入视频流。 - "-c:a", "aac"：音频编码格式为AAC。 - "-c:v", "copy"：视频流直接复制，不重新编码。 - "-f", "segment"：输出格式为分段。 - "-segment_time", String.valueOf(duration)：每个视频段的持续时间。 - "-max_muxing_queue_size", "1024"：设置复用缓冲队列的大小为1024。 - "-reset_timestamps", "1"：每个分段重置时间戳。 - workDir + "/%d.mp4"：输出文件的命名格式。

4. separateAudio(String videoPath, String outputPath)

整体作用：从视频文件中分离音频，并保存为MP3格式的音频文件。

参数解释： - videoPath：视频文件路径。 - outputPath：音频文件输出路径。 - "-hide_banner"：隐藏FFmpeg的启动横幅信息。 - "-y"：自动覆盖输出文件。 - "-i", videoPath：指定输入视频文件。 - "-f", "mp3"：输出格式为MP3。 - "-ab", "16k"：音频比特率为16kbps。 - "-ar", "12000"：音频采样率为12000Hz。 - "-ac", "1"：音频通道数为1（单声道）。

5. separateVideo(String videoPath, String outputPath)

整体作用：从视频文件中分离视频，并保存为指定路径。

参数解释： - videoPath：视频文件路径。 - outputPath：视频文件输出路径。 - "-hide_banner"：隐藏FFmpeg的启动横幅信息。 - "-y"：自动覆盖输出文件。 - "-i", videoPath：指定输入视频文件。 - "-an"：禁用音频流。 - "-vcodec", "copy"：视频流直接复制，不重新编码。

6. silenceAudio(double duration, String outPath)

整体作用：生成指定时长的静音音频文件。

参数解释： - duration：静音音频的持续时间（秒）。 - outPath：静音音频文件输出路径。 - "-hide_banner"：隐藏FFmpeg的启动横幅信息。 - "-f", "lavfi"：输入格式为libavfilter虚拟设备。 - "-i", "aevalsrc=0:s=44100:d=" + duration：生成44100Hz采样率，指定时长的静音音频。

7. combineAudio(List<Map<String, String>> audioList, String outputPath)

整体作用：合并多个音频文件。

参数解释： - audioList：包含多个音频文件路径及其起始时间的列表。 - outputPath：合并后的音频文件输出路径。 - cmd和filterComplex用于生成FFmpeg命令： - 添加输入音频文件。 - 生成复杂过滤器，以指定延迟时间合并音频。

8. changeSpeed(String audioPath, double speed, String outputPath)

整体作用：改变音频文件的播放速度。

参数解释： - audioPath：音频文件路径。 - speed：目标速度。 - outputPath：修改速度后的音频文件输出路径。 - "-y"：自动覆盖输出文件。 - "-i", audioPath：指定输入音频文件。 - "-filter:a", "atempo=" + speed：调整音频速度。

9. combineAudioAndVideo(String audioPath, String videoPath, String outputPath)

整体作用：合并音频和视频文件。

参数解释： - audioPath：音频文件路径。 - videoPath：视频文件路径。 - outputPath：合并后的音视频文件输出路径。 - "-y"：自动覆盖输出文件。 - "-i", videoPath：指定输入视频文件。 - "-i", audioPath：指定输入音频文件。 - "-c:v", "copy"：视频流直接复制，不重新编码。 - "-c:a", "aac"：音频编码格式为AAC。 - "-strict", "experimental"：使用实验性的AAC编码器。

10. pushStreamByPipe(String streamUrl)

整体作用：将输入流通过管道推送到指定的流媒体服务器。

参数解释： - streamUrl：流的URL。 - XArray.concat方法连接以下数组： - "-re"：以实时模式读取输入流。 - "-i", "-"：输入从标准输入读取。 - "-b:v", "3000k"：设置视频比特率为3000kbps。 - "-bufsize", "6000k"：设置缓冲区大小为6000kb。 - "-c:v", "copy"：视频流直接复制，不重新编码。 - "-c:a", "aac"：音频编码格式为AAC。 - "-reconnect", "1"：启用重新连接。 - "-reconnect_at_eof", "1"：在到达文件末尾时重新连接。 - "-reconnect_streamed", "1"：重新连接流。 - "-reconnect_delay_max", "3"：重新连接的最大延迟为3秒。

11. duration(String path)

整体作用：获取视频文件的时长。

参数解释： - path：视频文件路径。 - "-hide_banner"：隐藏FFmpeg的启动横幅信息。 - "-v", "error"：仅显示错误信息。 - "-show_entries", "format=duration"：显示格式中的时长信息。 - "-of", "default=noprint_wrappers=1:nokey=1"：输出格式为默认，但不打印包装器，并且不显示键。

12. convertToTs(String videoPath, String outputPath)

整体作用：将视频文件转换为TS格式。

参数解释： - videoPath：视频文件路径。 - outputPath：转换后的TS文件输出路径。 - "-hide_banner"：隐藏FFmpeg的启动横幅信息。 - "-y"：自动覆盖输出文件。 - "-i", videoPath：指定输入视频文件。 - "-c", "copy"：视频流和音频流直接复制，不重新编码。 - "-bsf:v", "h264_mp4toannexb"：对H.264视频流应用比特流过滤器。 - "-f", "mpegts"：输出格式

为MPEG-TS。

13. convertToMp4(String videoPath, String outputPath)

整体作用：将视频文件转换为MP4格式。

参数解释： - videoPath：视频文件路径。 - outputPath：转换后的MP4文件输出路径。 - "-hide_banner"：隐藏FFmpeg的启动横幅信息。 - "-y"：自动覆盖输出文件。 - "-i", videoPath：指定输入视频文件。 - "-c", "copy"：视频流和音频流直接复制，不重新编码。 - "-movflags", "faststart"：将MOOV原子放在文件开头，以便于快速启动。 - "-f", "mp4"：输出格式为MP4。

14. fastMov(String videoPath, String outputPath)

整体作用：将视频文件转换为快速启动的MOV格式。

参数解释： - videoPath：视频文件路径。 - outputPath：转换后的MOV文件输出路径。 - "-y"：自动覆盖输出文件。 - "-i", videoPath：指定输入视频文件。 - "-c", "copy"：视频流和音频流直接复制，不重新编码。 - "-movflags", "faststart"：将MOOV原子放在文件开头，以便于快速启动。

15. extractFirstFrame(String videoPath, String outputPath)

整体作用：从视频文件中提取第一帧并保存为图像文件。

参数解释： - videoPath：视频文件路径。 - outputPath：提取的图像文件输出路径。 - "-hide_banner"：隐藏FFmpeg的启动横幅信息。 - "-y"：自动覆盖输出文件。 - "-i", videoPath：指定输入视频文件。 - "-frames:v", "1"：仅提取一帧图像。
