第一章

  1. 分布式操作系统定义

    分布式系统是独立计算机的集合，在其用户看来是一个单一的连贯系统。

    由位于联网计算机上的硬件或软件组件组成的任何系统，这些组件通过消息传递来通信和协调它们的操作。

  2. 特性

    1. Hidden ：Hide the difference and the communication among all kinds of computers
    2. Consistent and Uniform
    3. Scalability ：伸缩性，即使坏了一部分整个系统也不会瘫痪（允许某些功能不可用了）
    4. Concurrency ：Coordination among different concurrently executing programs that share resource
    5. No Global Clock ：

  3. 目标

    1. Making Resources Accessible

    2. Distribution Transparency 看起来只有一个电脑

    3. Openness

      Mechanism : (How to do it)It need cache

      Policy : (What will be done)Size? • Consistency? • Always or only one session? • Can user control?

    4. Scalability

第二章 Communication

  1. UDP offers no guarantee of delivery 不可靠的传输 No setup costs 不需要建立连接，额外花销小

  2. RPC(Remote Procedure Call)

    Problems:

    1. Call by Value：异构的计算机对于相同的字符串的解读不同
    2. Call by Reference：
      1. For array: copy/restore 把整个数组发过去，搞完之后再发回本机修改
      2. For ordinary pointer 接收机器拿着指针回来找
    3. How to Create Stub：Interface Definition Language (IDL)使用统一接口

  3. RMI（Remote Method Invocation）

    Client:Proxy

    Server:Skleton

    RMI and RPC are same theoretically.

  4. 同步通信收发都是堵塞的，异步通信发是非堵塞的收都可以

  5. Transient communication:A message is not stored in communication system 瞬时通信不存储信息

    1. Sockets Interface
      1. TCP/UDP
      2. IP Multicast: IP multicast is available only via UDP
    2. MPI(Message Passing Interface standard)

  6. Persistent Communication:A message is stored in communication system until it is sent to receiver 持久通信

    1. Message Queue Model 保证一定送达但是不保证时间长短
      1. router：进行接收转发
      2. relay：接收之后进行处理 Relay can build a scalable message-queuing system
      3. Application of Message Queue System
        1. Asynchronization 异步
        2. Decoupling of application 解耦
        3. Flow Cutting 流切割

  7. Stream Oriented Communication

    1. Continuous representation media: Temporal relationship between data items is IMPORTANT to explain the meaning of data.

    2. Discrete representation media: Temporal relationship between data items is NOT IMPORTANT to explain the meaning of data

    3. Transmission types

      1. Asynchronous transmission mode

        Data items are transmitted one by one. No time restriction on transmission.

      2. Synchronous transmission mode

        Every data item has a maximum end-to-end delay defined for each unit in a data stream.

      3. Isochronous transmission mode

        Data item must be transmitted on time.

    4. Interleaving frames 交错帧，减小数据包丢失的影响

    5. QoS Negotiation是一种网络协议，用于协商和确定网络中各种流量的优先级、带宽、延迟等服务质量参数。

第三章

  1. Server Classification

    1. Iterative Server：Server is responsible for dealing with request and return the result to client
    2. Concurrent Server：Server doesn’t deal with request from client. It sends the request to a thread or a process,then wait for next request

  2. Out-of-band data ： data that is to be processed by the server before any other data from that client.优先级很高，需要及时处理

  3. Stateless Server：Don’t save any information about client

    Server with state：Maintain information about client

  5. Code Migration

    1. Process migration ：An entire process was moved from one machine to another.

      1. Weak Mobility: Only migrate code segment and some initialization data

      2. Strong Mobility: Migrate the code segment and the executing segment

    2. Resource Migration:

      1. Process-Resource Binding: ID,Value,Type
      2. Resource-Machine Binding: Unattached Resource,Fastened Resource,Fixed Resource

  7. Processor Allocation Algorithm

    1. Deterministic Algorithm：Know all information about all processes
    2. Heuristic Algorithm：The load of a system is unpredictable.

  1. Flat Naming
  2. Struct Naming

    2. Recursive Name Resolution：递归名称解析：主机只发一次，服务器之间进行通讯完成工作
  3. Attribute-Based Naming：To describe an entity in terms of (attribute,value) pairs

第五章

  1. 物理时钟

    1. Cristian Algorithm
      1. 如果有一个有WWV接收器的机器，其他机器都可以向这个机器请求时间
      2. Time is not allowed to run backward.如果一个机器的时间在同步时钟之后，时间回溯是不被允许的
    2. Berkeley Algorithm
      1. Time server polls the time of every machine
      2. Average the results and tell the time to other machine
    3. Averaging Algorithm
      1. Every machine broadcast its time to others in every time interval
      2. After broadcasting,machine collect time response from other machines in an interval S
      3. Average all the time values，Discard m highest and m lowest
      4. Try to add an estimated time delay to correct the time
    4. 应用: Network Time Protoc ol (NTP)

  2. 逻辑时钟

    1. 为什么不用物理时钟

    2. Lamport’s Logical Clocks

      1. If a and b are two events in a **same process**,and a happens before b,then a→b is true
    2. If a is a **sending** event of a message in a process,b is receiving event of this message in another process,then a→b is true
    3. If a→b is not true,b→ a is not true,a and b is concurrent 并发

  3. Vector Clocks

  1. Election Algorithm

    1. Bully Algorithm

    进程P发现协调员(coordinator)无法工作，发起一个选举，给所有比自己PID大的进程Q发消息，要是应答了则P放弃，Q继续上述过程，直到只剩一个，成为新的协调员

    1. Ring Algorithm

      逻辑上的环，每个进程知道自己的后继进程。转一圈之后选PID最大的当新协调员。

  2. Mutual Exclusion

    1. Centralized Algorithm

      P是协调者，Q想进去要问P，能进就可以，不能进Q就进等待队列。里面的进程出来之后授权队列头进程

    2. Distributed Algorithm

      P想进去，发请求给所有进程Q，要是Q不想进去就发ok回来，要是Q就在里面就不回应，要是Q也想进去就比较时间戳，要是比自己早就回ok

    3. Token Ring Algorithm

      一个令牌在环里绕，拿到令牌的可以进去。

      这个算法不是一个fair algorithm，前面俩都是fair algorithm

  3. Distributed Transaction

    A transaction can • Protect shared resource from concurrent accesses • Permit a process read and write several data items as if it is read and written by a single atomic operation

    1. Characteristics

      1. Atomic • A transaction cannot be separated.

      2. Consistent • A transaction cannot spoil the consistent item of system.

      3. Isolated • Concurrent transaction cannot affect each other.

      4. Durable • Once a transaction is committed,the change will be permanent.

      5. RollBack

      6. Private Workspace: Give a private workspace to a transaction when it begins

        All data are read and written in the private workspace until the transaction is committed or failed

    2. Writeahead Log

      1. Before a file is updated,a record is written in a log

      2. Only after write the log successfully,the change can be committed to the file

        1. Serial Equivalence

    3. Several transactions can be executed at the same time but the final result is the same as the situation that all transactions are executed one by one

    4. Two Concurrent Control Methods

      1. Pessimistic approaches 不允许可能出现的问题

        Timestamp Ordering

        1. A transaction’s request to write an object is valid only if that object was last read and written by earlier transactions
        2. A transaction’s request to read an object is valid only if that object was last written by earlier transactions

      2. Optimistic approaches 可以有问题，回滚就行了

第六章

  1. Replication 副本

    1. 为什么需要副本

      • Reliability

      • Performance

      • Fault tolerance

    2. CAP

      1. Partition Tolerance 分区容错性
        1. Every subnetwork is called a partition
        2. The system still work when communication fail
      2. Consistency 一致性
        1. All replica in the system should be consist
      3. Availability 可用性
        1. All nodes in the system can be used when user need it
      4. 上述三个不能bingcun
        1. 如果想有P，那肯定得有预留的副本，那么一致性和全可用性就不可能实现
        2. 如果想有CA,那P肯定不能实现

    3. 一致性

      1. Strict Consistency
        1. Any read operation on data item x can return back the result of last write on x.
        2. Depend on absolute global time
      2. Sequential Consistency
        1. The result of all process’s read and write operation are the same as if they are executed according to a specific order.
        2. 一个进程内部执行顺序一致就行，俩进程谁先谁后无所谓？
      3. Linearizability
        1. every operation has a timestamp,and satisfies following condition
        2. 时间在前的先发生
      4. Causal Consistency
        1. Writes that are potentially causally related must be seen by all processes in the same order.
        2. 得分析前面的潜在因果关系，要是俩写操作判断不出来谁前谁后，那读的时候谁前谁后都可以

    4. 修改的传播方法

      1. Only propagate a notification of an update

        告诉别人我已经改了，别再用那个旧的了

      2. Transfer data from one copy to another

        把改了的数据发给别人

      3. Propagate the update to other copies

        告诉别人我怎么改的

    5. 传染病模型

      1. 传统的方法是P一个人告诉其他所有人更新消息，传染病模型是P告诉Q之后，PQ一起告诉别人

      2. Anti-Entropy Model 反熵模型

        A server P select another server Q randomly and exchange updates

      3. Gossiping Model 八卦模型

        P拿到更新之后随机再告诉别人要更新
